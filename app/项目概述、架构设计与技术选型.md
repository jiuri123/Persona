### 项目概述、架构设计与技术选型

#### 1. 项目概述

本项目 "Persona" 是一个创新的 AI 社交应用。其核心是让用户创建、培养一个由 AI 驱动的“数字人格” (Persona) 。

核心基础功能包括：

1. **Persona 创作**：用户可以自定义（名称、头像、故事）或通过 AI 辅助生成一个 Persona。
2. **Persona 社交**：
   - **社交广场**：Persona 可以（由用户触发）发布图文动态到公共信息流。用户可以浏览并关注其他 Persona。
   - **直接对话**：用户可以和*任何* Persona 发起 1v1 聊天，Persona 会根据其人设（通过云端 LLM API）进行回复。
3. **Persona 共生**：用户可以在一个“私聊”界面与*自己的* Persona 对话，以此来引导和调整它。

#### 2. 客户端架构设计：MVVM + 仓库 (Repository) 模式

使用 Android 官方推荐的 **MVVM (Model-View-ViewModel)** 架构。

- **View (视图层)**:  `Activity` 和 `Fragment` (使用 XML 布局)。它们只负责展示 UI 和发送用户事件（如点击）。
- **ViewModel (视图模型)**: 专门的类，用于存储和管理与 UI 相关的数据。它独立于 UI，因此在屏幕旋转等情况下数据不会丢失。View 通过**LiveData**来“订阅” ViewModel 中的数据变化，实现自动更新。
- **Model (模型层)**: 负责应用的数据和业务逻辑。在这里使用 **Repository (仓库) 模式**。

#### 3. 技术选型 (Tech Stack)

| **类别**          | **技术选型**             | **为什么选择它？**                                           |
| ----------------- | ------------------------ | ------------------------------------------------------------ |
| **开发语言**      | **Java**                 | 我已学过 Java 语法。虽然 Kotlin 是 Android 的首选，但在 10 天内同时学习新语言、新框架和 AI 是不现实的。用我已有的 Java 基础 能最快地产出。 |

| **核心组件**      | **Activity / Fragment**  | `Activity` 作为页面的主容器，`Fragment` 用于在 Activity 内切换不同的视图（例如，用底部导航栏切换“广场”、“私聊”等页面）。 |
| **列表展示**      | **RecyclerView**         | 无论是“社交广场”还是“聊天界面”，都需要用它来高效显示可滚动的列表。 |

| **架构组件**      | **ViewModel + LiveData** | MVVM 架构的核心，用于实现数据驱动的 UI 和解耦。              |
| **网络请求**      | **Retrofit 2**           | 行业标准。用于调用云端大语言模型 API 19 和（进阶中）的真实后端 API。我学过 Spring，它和 Spring 中定义 API 接口有些相似。 |
| **JSON 解析**     | **Gson**                 | 配合 Retrofit 使用，自动将服务器返回的 JSON 字符串转换为 Java 对象。 |
| **图片加载**      | **Glide**                | 负责高效加载网络图片（如头像、动态图片）并处理缓存和占位符。 |
| **数据源 (基础)** | **Mock 数据**            | 按要求，先用 Java 的 `ArrayList`, `HashMap` 等在代码中写死数据，保证核心流程跑通 20。 |

#### 4. 优先攻克的进阶挑战

我的设计已经为以下两个进阶挑战打下了坚实基础：

1. [项目架构优化] 从 Mock 走向真实服务 (15分) 
   - **实现**：采用的 Repository 模式就是为这个挑战设计的。基础版完成后，你可以快速增加一个 `LocalDataSource`（使用 **Room 数据库**，这是 Android 官方推荐的本地数据库）来实现多账户登录和数据隔离。
2. [功能体验优化] 富文本 (Markdown) (5分) 
   - **实现**：当 Persona 回复的文本需要展示时，引入一个 Markdown 渲染库（如 **Markwon**），用它来处理文本并显示在 `TextView` 上。这是一个投入产出比较高的加分项。
